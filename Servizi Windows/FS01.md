FS01: file server SMB, permessi NTFS, cartelle reparto, quote, versioning, auditing

### Ruolo

#### Obiettivo
FS01 ospita le condivisioni file aziendali del gruppo alberghiero. Il servizio permette agli utenti autorizzati di accedere ai dati di reparto e ai documenti condivisi tra le tre strutture usando credenziali Active Directory.

Il file server centralizza i dati nel CED de Il Sogno, evitando copie locali non controllate nelle singole sedi e semplificando backup, permessi, auditing e manutenzione.

#### Posizionamento
FS01 sarà eseguito come macchina virtuale sul primo host ESXi, insieme a DC01, NPS01 e al server gestionale.

Le sedi remote, La Meraviglia e L'Incanto, accederanno a FS01 tramite VPN site-to-site verso il CED. Il traffico SMB deve essere permesso solo dalle VLAN aziendali autorizzate, non dalle reti guest, Smart TV, tablet camere o sale meeting.

### Indirizzamento

#### Rete Server/CED
FS01 si trova nella rete Server/CED de Il Sogno, secondo il piano di indirizzamento della sede.

- Sede: Il Sogno
- VLAN: 11 - Server_CED
- Rete: `10.30.11.0/27`
- Gateway: `10.30.11.1`

| Server | FQDN | Indirizzo IP | Ruoli principali |
|---|---|---:|---|
| FS01 | `fs01.corp.ilsogno.it` | `10.30.11.12` | File server SMB, cartelle reparto, permessi NTFS |

L'indirizzo IP e il record DNS sono statici. FS01 deve risolvere correttamente DC01 e DC02, perché autenticazione, gruppi e permessi dipendono da Active Directory.

### Condivisioni SMB

#### Obiettivo
FS01 espone condivisioni SMB per reparti, sedi e area gestionale comune. L'accesso avviene tramite gruppi Active Directory, non tramite permessi assegnati direttamente ai singoli utenti.

#### Condivisioni previste
Le condivisioni definitive saranno rifinite in base al gestionale e alle esigenze operative, ma la struttura proposta è:

| Condivisione | Percorso logico | Uso previsto |
|---|---|---|
| `\\fs01\Direzione` | Dati direzione e documenti riservati | Direzione e direzione generale |
| `\\fs01\Contabilita` | Dati amministrativi e contabili | Contabilità unica |
| `\\fs01\Reception` | Documenti operativi reception | Reception delle strutture |
| `\\fs01\IT` | Documentazione tecnica e strumenti IT | Personale IT |
| `\\fs01\Sicurezza` | Documenti e report sicurezza | Addetti sicurezza |
| `\\fs01\Eventi` | Materiale sale meeting ed eventi | Marketing/eventi e personale autorizzato |
| `\\fs01\Condivisa_Gruppo` | Area comune inter-hotel | Direzione, contabilità, reception e ruoli autorizzati |

Le cartelle personali degli utenti possono essere previste in una fase successiva, ma non sono essenziali per la prima implementazione.

### Permessi NTFS

#### Modello di accesso
I permessi saranno gestiti con gruppi Active Directory. La regola di base è: utenti nei gruppi, gruppi applicati alle cartelle.

| Cartella | Gruppi con accesso |
|---|---|
| Direzione | `GG_Direzione`, `GG_DirezioneGenerale`, `GG_IT` |
| Contabilita | `GG_Contabilita`, `GG_DirezioneGenerale`, `GG_IT` |
| Reception | `GG_Reception`, `GG_Direzione`, `GG_IT` |
| IT | `GG_IT` |
| Sicurezza | `GG_Sicurezza`, `GG_Direzione`, `GG_IT` |
| Eventi | `GG_Eventi`, `GG_Marketing`, `GG_Direzione`, `GG_IT` |
| Condivisa_Gruppo | gruppi autorizzati secondo necessità operativa |

Gli utenti normali non devono avere privilegi amministrativi sul file server. Le modifiche ai permessi devono essere gestite dall'IT tramite gruppi AD.

#### Separazione dei dati
Ogni reparto vede solo le cartelle necessarie al proprio lavoro. La direzione generale e la contabilità possono accedere ai dati inter-hotel secondo necessità, come richiesto dalla gestione centralizzata del gruppo.

### Quote e versioning

#### Quote
FS01 userà quote per evitare crescita incontrollata dei dati e saturazione dello storage.

Le quote saranno applicate almeno a:
- cartelle di reparto;
- area condivisa inter-hotel;
- eventuali cartelle personali future;
- repository temporanei o aree di scambio file.

Le soglie definitive dipendono dalla dimensione dello storage assegnato alla VM e dal NAS/backup appliance. Il documento di budget e storage definirà i valori numerici.

#### Versioning
Il versioning consente di recuperare versioni precedenti dei file in caso di errore umano, sovrascrittura o cancellazione accidentale.

La soluzione prevista include:
- shadow copies o funzionalità equivalenti lato Windows Server;
- snapshot lato storage/NAS dove disponibile;
- backup periodico gestito da BK01/NAS.

Il versioning non sostituisce il backup: serve per recuperi rapidi, mentre BK01/NAS gestisce ripristini completi e scenari di guasto.

### Auditing

#### Obiettivo
L'auditing consente di tracciare accessi e modifiche ai dati sensibili. La funzione è utile soprattutto per cartelle contabili, direzione, sicurezza e documenti inter-hotel.

#### Eventi da tracciare
L'auditing sarà abilitato in modo selettivo, per evitare log troppo rumorosi.

Eventi consigliati:
- accesso negato a cartelle riservate;
- modifica o cancellazione di file sensibili;
- cambio permessi su cartelle condivise;
- accesso amministrativo al file server;
- errori ripetuti di autenticazione.

I log dovranno essere consultabili dall'IT e, se previsto dalla soluzione premium, integrabili con monitoring centralizzato.

### Sicurezza e rete

#### Accesso consentito
FS01 deve essere raggiungibile solo dalle VLAN aziendali autorizzate e tramite VPN per le sedi remote.

Accesso consentito:
- VLAN staff e uffici;
- VLAN amministrazione;
- VLAN direzione;
- VPN remota per IT, direzione e contabilità autorizzati;
- server interni che necessitano accesso a file condivisi.

Accesso non consentito:
- guest Wi-Fi;
- reti camere;
- Smart TV;
- tablet camere non aggiunti al dominio;
- sale meeting ospiti;
- reti IoT non autorizzate.

#### Dipendenze
FS01 dipende da:
- DC01/DC02 per autenticazione e gruppi AD;
- DNS interno per risoluzione `fs01.corp.ilsogno.it`;
- VPN site-to-site per accesso dalle sedi remote;
- BK01/NAS per backup e snapshot;
- policy di sicurezza e hardening endpoint.

### Backup e ripristino

#### Obiettivo
FS01 contiene dati aziendali e deve essere incluso nelle procedure di backup del CED.

La strategia prevista include:
- backup completo della VM;
- backup dei dati condivisi;
- snapshot periodici;
- retention coerente con le esigenze operative;
- copia offsite o cloud tramite BK01/NAS.

#### Ripristino
Devono essere previste procedure per:
- ripristino singolo file;
- ripristino cartella;
- ripristino permessi NTFS;
- ripristino completo della VM FS01;
- verifica periodica dei restore.

### Test di accettazione

- Un utente di reception accede solo alla condivisione autorizzata.
- Un utente di contabilità accede alla cartella `Contabilita` e alla parte gestionale autorizzata.
- Un utente non autorizzato riceve accesso negato su cartelle riservate.
- Un client staff accede a `\\fs01\Condivisa_Gruppo` usando nome DNS, non indirizzo IP.
- Un client guest non raggiunge le condivisioni SMB.
- Una sede remota accede a FS01 attraverso VPN site-to-site.
- Una versione precedente di un file può essere recuperata.
- Un file cancellato può essere ripristinato da backup o snapshot.
- Gli eventi di accesso negato o modifica sensibile vengono registrati.
