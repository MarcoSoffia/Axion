DC02: ridondanza dominio, DNS, Global Catalog, DHCP failover

### AD

#### Nome dominio Active Directory
DC02 appartiene allo stesso dominio Active Directory creato da DC01:

- Nome DNS dominio: `corp.ilsogno.it`
- Nome NetBIOS: `ILSOGNO`
- Forest root domain: `corp.ilsogno.it`

DC02 non crea un dominio separato: viene promosso come Domain Controller aggiuntivo del dominio esistente.

#### Indirizzamento AD
DC02 si trova nella rete Server/CED de Il Sogno, secondo il piano di indirizzamento della sede.

- Sede: Il Sogno
- VLAN: 11 - Server_CED
- Rete: `10.30.11.0/27`
- Gateway: `10.30.11.1`

| Server | FQDN | Indirizzo IP | Ruoli principali |
|---|---|---:|---|
| DC01 | `dc01.corp.ilsogno.it` | `10.30.11.10` | AD DS, DNS, DHCP, Global Catalog |
| DC02 | `dc02.corp.ilsogno.it` | `10.30.11.11` | AD DS, DNS, DHCP failover, Global Catalog |

Gli IP sono statici e non devono essere inclusi in nessun pool DHCP. Gli scope DHCP delle reti aziendali distribuiranno `10.30.11.10` e `10.30.11.11` come DNS primario e secondario.

#### AD addresses
Gli indirizzi Active Directory da usare nella configurazione di rete sono:

| Uso | Primario | Secondario |
|---|---:|---:|
| Domain Controller | `dc01.corp.ilsogno.it` | `dc02.corp.ilsogno.it` |
| DNS aziendale | `10.30.11.10` | `10.30.11.11` |
| DHCP server | `10.30.11.10` | `10.30.11.11` |
| Global Catalog | `dc01.corp.ilsogno.it` | `dc02.corp.ilsogno.it` |

Questi indirizzi devono essere raggiungibili dalle VLAN aziendali attraverso routing locale o VPN site-to-site. Le reti guest, Smart TV, tablet non aggiunti al dominio e sale meeting non devono usare direttamente questi indirizzi come DNS.

### Domain Services

#### Obiettivo
DC02 garantisce ridondanza per il dominio `corp.ilsogno.it`. In caso di manutenzione o indisponibilità di DC01, DC02 consente agli utenti di continuare ad autenticarsi e ai client di continuare a risolvere il dominio.

DC02 sarà installato sul secondo host ESXi, separato fisicamente da DC01, così da ridurre il rischio che un guasto hardware fermi tutti i servizi di dominio.

#### Ruolo di DC02
DC02 avrà il compito di:
- replicare Active Directory con DC01;
- autenticare utenti e computer se DC01 non è disponibile;
- mantenere una copia aggiornata di utenti, gruppi, computer e unità organizzative;
- applicare le Group Policy ai client aziendali insieme a DC01;
- fornire servizi di dominio a file server, NPS, VPN e gestionale;
- aumentare la resilienza del CED durante manutenzione o failure di un host.

DC02 non detiene inizialmente i ruoli FSMO del dominio e della foresta. Tali ruoli restano su DC01, salvo trasferimento manuale in caso di manutenzione pianificata o recovery.

#### Replica Active Directory
La replica tra DC01 e DC02 mantiene coerenti utenti, gruppi, computer, password, Group Policy e configurazione del dominio.

La replica deve essere monitorata perché eventuali errori possono causare:
- login non coerenti tra Domain Controller;
- policy non aggiornate sui client;
- gruppi o permessi non allineati;
- problemi DNS sulle zone integrate in Active Directory.

#### Test di accettazione
- DC02 risulta Domain Controller del dominio `corp.ilsogno.it`.
- Un utente riesce ad accedere a un PC aggiunto al dominio usando DC02.
- Le Group Policy risultano disponibili anche con DC01 non raggiungibile.
- La replica Active Directory tra DC01 e DC02 non segnala errori.
- Spegnimento DC01: login e risoluzione dominio continuano tramite DC02.

### DNS

#### Obiettivo
DC02 ospita il servizio DNS interno in modalità ridondante. Le zone DNS sono integrate in Active Directory e replicate tra DC01 e DC02.

Questo permette ai client aziendali di continuare a risolvere il dominio, i Domain Controller e i servizi interni anche se DC01 non è disponibile.

#### Architettura
- DC01: DNS primario integrato con Active Directory.
- DC02: DNS ridondante integrato con Active Directory.
- Zone DNS: replicate tramite Active Directory.
- Client aziendali: ricevono DC01 e DC02 come DNS tramite DHCP.
- Reti guest e dispositivi non aziendali: usano resolver locale del firewall o DNS pubblici filtrati.

#### Zone DNS
DC02 replica la zona DNS interna principale:

- Zona interna: `corp.ilsogno.it`
- Record DC01: `dc01.corp.ilsogno.it`
- Record DC02: `dc02.corp.ilsogno.it`
- File server: `fs01.corp.ilsogno.it`
- NPS/RADIUS: `nps01.corp.ilsogno.it`
- Backup/NAS: `bk01.corp.ilsogno.it`
- CMS multimediale: `media01.corp.ilsogno.it`
- Server gestionale: `gestionale01.corp.ilsogno.it`

Le zone reverse lookup delle subnet aziendali saranno replicate anche su DC02.

#### Risoluzione DNS
```text
Client aziendale
    -> DNS DC01 / DC02
    -> record interno AD o CED
    -> se record non interno, inoltro verso resolver esterno
```

Se DC01 non risponde, i client aziendali possono interrogare DC02 come DNS secondario.

#### Sicurezza e controllo
- Consentire query DNS a DC02 solo dalle VLAN autorizzate.
- Bloccare accesso DNS diretto a DC02 da guest Wi-Fi e reti non fidate.
- Replicare solo zone DNS integrate in Active Directory.
- Abilitare aggiornamenti dinamici DNS solo in modalità sicura.
- Monitorare eventi DNS critici e problemi di replica.

#### Test di accettazione
- Client aggiunto al dominio risolve `corp.ilsogno.it` usando DC02.
- Client staff accede a file server e gestionale usando nomi DNS.
- Client guest non può interrogare direttamente il DNS di DC02.
- Spegnimento DC01: la risoluzione interna continua tramite DC02.

### Global Catalog

#### Obiettivo
DC02 sarà configurato come Global Catalog Server. Il Global Catalog mantiene un indice degli oggetti principali di Active Directory e consente ricerche rapide di utenti, gruppi, computer e risorse.

Nel progetto Axion, configurare anche DC02 come Global Catalog riduce la dipendenza da DC01 durante login, ricerca oggetti e risoluzione dei gruppi.

#### Ridondanza
Con DC01 e DC02 entrambi Global Catalog, il dominio mantiene una copia consultabile degli oggetti principali anche durante manutenzione o indisponibilità di uno dei due Domain Controller.

#### Test di accettazione
- Ricerca utenti e gruppi funzionante da strumenti Active Directory su DC02.
- Login utente funzionante anche con DC01 non disponibile.
- Risoluzione appartenenza gruppi corretta per utenti di reception, direzione, contabilità e IT.

### DHCP Failover

#### Obiettivo
DC02 sarà configurato come partner DHCP failover di DC01. In caso di indisponibilità di DC01, DC02 continuerà a rilasciare lease DHCP per le VLAN aziendali gestite centralmente.

Il DHCP resta centralizzato nel CED, mentre i gateway delle VLAN rimangono locali alle sedi. La Meraviglia e L'Incanto inoltrano le richieste DHCP tramite DHCP relay sui firewall pfSense di filiale.

#### Architettura
- DC01: DHCP primario.
- DC02: DHCP failover.
- pfSense sedi: DHCP relay per VLAN.
- Gateway VLAN: locale alla sede.
- DNS distribuiti via DHCP per reti aziendali: `10.30.11.10`, `10.30.11.11`.
- DNS per reti guest, Smart TV, tablet non aggiunti al dominio e meeting: firewall locale o resolver filtrati.

#### Ambiti DHCP
Gli scope definitivi dipendono dal piano di indirizzamento della rete e saranno riportati nella documentazione di rete. In questo documento viene fissato il ruolo di DC02 come partner di failover.

| Sede | VLAN | Subnet | Gateway | Pool DHCP | DNS distribuiti |
|---|---|---|---|---|---|
| Il Sogno | Staff / uffici | Piano IP rete | Gateway VLAN locale | Pool aziendale | `10.30.11.10`, `10.30.11.11` |
| La Meraviglia | Staff / reception | Piano IP rete | Gateway VLAN locale | Pool aziendale | `10.30.11.10`, `10.30.11.11` |
| L'Incanto | Staff / reception | Piano IP rete | Gateway VLAN locale | Pool aziendale | `10.30.11.10`, `10.30.11.11` |
| Guest / meeting | Ospiti ed eventi | Piano IP rete | Gateway VLAN locale | Pool guest | DNS firewall locale |
| Smart TV / tablet | Dispositivi camere | Piano IP rete | Gateway VLAN locale | Pool dispositivi | DNS firewall locale |

#### Mini-case
Le mini-case usano DHCP locale sul router, con configurazione standardizzata e documentata centralmente. Questa scelta consente alla rete interna della mini-casa di continuare a funzionare anche se il tunnel WireGuard verso il CED non è disponibile.

#### Test di accettazione
- DC02 replica gli scope DHCP di DC01.
- Spegnimento DC01: un client staff riceve lease da DC02.
- Client aziendale riceve gateway locale e DNS `10.30.11.10`, `10.30.11.11`.
- Client guest riceve DNS del firewall locale e non raggiunge direttamente i Domain Controller.
- VPN sede remota attiva: DHCP relay verso DC01/DC02 funzionante.
- VPN sede remota down: comportamento documentato per le reti aziendali della sede.

### Backup e monitoraggio

DC02 sarà incluso nelle procedure di backup del CED. Il backup dovrà coprire almeno il System State del Domain Controller, così da permettere il ripristino dei servizi Active Directory in caso di guasto grave o errore amministrativo.

Il monitoraggio dovrà verificare:
- stato dei servizi AD DS, DNS e DHCP;
- replica tra DC01 e DC02;
- stato del Global Catalog;
- stato del DHCP failover;
- disponibilità degli scope DHCP;
- spazio disco e risorse della VM;
- eventi critici nei log di sistema e sicurezza.
